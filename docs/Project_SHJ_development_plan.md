# 游戏策划案：Project SHJ (代号：山海经)

文档版本： 1.0 (Concept Master)

核心类型： 开放世界 / 沙盒生存 / 宠物捕捉 / 模拟经营 / UGC社交

美术风格： High-Fidelity Voxel + LowPoly

目标平台： PC / Mobile (跨端互通，重点优化移动端触控体验)

对标竞品： Roblox (社交与物理交互) + Palworld (核心循环) + Clash of Clans (异步PVP/家园攻防)

---

## 第一章：核心概念 (High Concept)

### 1.1 一句话介绍 (Elevator Pitch)

“抓捕万物作为生产力，创造关卡挑战全世界。”

这是一个物理驱动的方块世界。玩家不仅要捕捉神奇的“灵兽”来建立全自动化工厂，还要利用积木和逻辑方块，将自己的基地改造成充满陷阱与守卫怪物的“副本（Dungeon）”，供其他玩家挑战并赚取收益。

### 1.2 核心体验支柱 (Core Pillars)

1. **物理驱动的幽默感 (The Roblox DNA)：**
    - 拒绝僵硬的动画。灵兽被扔出去会弹跳，工作累了会瘫倒，被传送带运送时会堆叠。这种“滑稽的物理反馈”是Roblox类游戏的核心魅力，能极大冲淡“压榨宠物”的道德负担，增加直播效果。
2. **所见即所得的自动化 (Visible Automation)：**
    - 灵兽不仅仅是数值加成，而是可视化的“活体机器”。火系狐狸是对着熔炉喷火，怪力熊是背着矿石在跑。看着基地从空地变成繁忙的流水线，是中期的核心爽点。
3. **社交即玩法 (PVP as UGC)：**
    - **“攻城战”Roblox化：** 玩家的基地不仅是生产中心，更是**“自定义关卡”**。通过布置陷阱、炮台和驻守灵兽，玩家创造属于自己的“Obby (障碍赛跑)”或“Boss Rush”关卡。

## 第二章：用户旅程与心流设计 (User Journey & Flow)

为了留存不同类型的玩家，我们将体验划分为三个阶段：**生存(Survival) -> 经营(Tycoon) -> 创造(Creator)**。

### 2.1 新手期 (0-2小时)：Hook (钩子)

- **情境：** 玩家降落在初始空岛，只有一把“物质枪”。
- **痛点：** 砍树慢，打怪难，肚子饿。
- **AHA Moment (顿悟时刻)：**
    - 玩家费力抓住了第一只“方块史莱姆”。
    - 无意中把史莱姆扔向农田，它竟然自动开始浇水！
    - **心理反馈：** “原来我不必自己干活！” —— 瞬间建立了对捕捉的极度渴望。

### 2.2 成长期 (2-20小时)：Habit (习惯)

- **情境：** 需要大量金属和电力来制造高级抓捕球和武器。
- **玩法循环：**
    - 缺铁 -> 去火山抓“熔岩龟” -> 扔进矿场自动产铁。
    - 缺电 -> 去雷鸣废墟抓“电击鼠” -> 连接发电机。
- **心流控制：** 当玩家觉得手动搬运资源很累时，适时解锁“传送带”和“自动化物流”，释放压力，获得极大的成就感。

### 2.3 终局 (20h+)：Hobby (爱好/社交)

- **情境：** 资源溢出，神兽在手。
- **玩法：** **“道馆主模式”**。
    - 玩家利用多余的资源构建一个巨大的迷宫。
    - 放置“逻辑方块”（如：踩踏板 -> 释放滚石）。
    - 将自己培养的最强神兽设定为守关BOSS。
    - 发布到全服大厅。其他玩家挑战成功拿资源，失败则留下“门票”。

# 第三章：核心系统深度拆解 (System Mechanics - Detailed Expansion)

本章节重点阐述游戏的核心循环：**“资源获取（地图/农业） -> 产能转化（灵兽工作流） -> 价值验证（战斗/塔防）”**。

## 3.1 灵兽经营与农场自动化循环 (Farming & Automation Loop)

在本项目中，灵兽不仅仅是战斗伙伴，更是**“可视化的自动化机器”**。不同于传统模拟经营的“菜单点选”，本作强调**“物理交互”**（抓举、投掷）与**“多阶段工作流”**。

### 3.1.1 农业与畜牧工作流 (The Biological Loop)

这是游戏早期的核心，用于解决“饥饿”并产出基础材料。

1. **种植循环 (Farming Cycle)**
    - **Phase 1 播种 (Seeding):** 需`[🌱播种]`技能。灵兽（如叶泥泥）在田地格子上蹦跳，喷洒像素种子。
    - **Phase 2 浇水 (Watering):** 需`[💧浇水]`技能。灵兽（如喷壶象）对着作物喷水，进度条涨满后作物开始生长。
    - **Phase 3 收割 (Harvesting):** 需`[🖐采集]`或`[🌾收割]`技能。灵兽攻击成熟作物，作物变为**物理掉落物**。
    - **Phase 4 搬运 (Transporting):** 需`[📦搬运]`技能。**这是自动化的关键闭环**。灵兽将掉落物拾起，运送到`[饲料箱]`或`[冰箱]`。
    - *痛点设计：* 如果没有搬运工，作物会在5分钟后腐烂消失，强迫玩家配置完整的“灵兽工作队”。
2. **牧场放牧 (Ranching)**
    - **机制：** 将特定灵兽扔进“牧场围栏”，它们不再移动，而是定期在脚下生成副产品。
    - **产出表：**
        - `棉花羊` -> **羊毛** (布料/防具基础)。
        - `炸弹鸟` -> **火药** (子弹基础)。
        - `熔岩蜥` -> **火焰腺体** (生火燃料)。
        - `宝石兽` -> **金币/钻石** (极低概率)。

### 3.1.2 消耗品与Buff体系

产出的农产品通过**烹饪锅**加工，产生三种价值：

- **饱食度 (Hunger):** 维持灵兽工作不饿死。
- **San值 (Sanity):** 心理健康。吃好吃的能恢复San值；San值过低灵兽会**罢工**（原地睡觉）或**暴动**（破坏设施）。
- **工作/战斗Buff:**
    - *大力生菜:* 10分钟内工作速度+30%。
    - *岩石硬皮饼:* 10分钟内防御力+20%。

## 3.2 灵兽分类与全图鉴设计 (V-Pet Roster & Stats)

### 3.2.1 定位与词条系统 (Traits)

灵兽分为战斗型（高HP/攻/防）和生产型（高工作技能等级、低食量）。

灵兽必须设计为**“模块化”**的，方便美术量产和换皮。

- **属性维度：**
    - **战斗：** HP / 攻 / 防 / 元素（水火草雷土风暗）。
    - **工作适应性 (Work Suitability)：** 这是一个图标矩阵。例如 `[生火 Lv3]` `[搬运 Lv1]`。
    - **San值 (Mood)：** 长期工作或环境恶劣会降低。归零后灵兽会“罢工”（如原地睡觉、破坏设施）。
- **词条系统 (Traits)：** 随机生成的被动技能，决定它是“极品打工仔”还是“战斗神兽”。
    - *工匠精神：* 工作速度+50%，攻击力-20%。
    - *脑肌：* 攻击力+30%，工作速度-20%。

每只灵兽生成时随机获得 0-4个词条，决定其最终用途。

## 3.3 世界地图与资源分布 (Map & Biomes)

地图设计采用**“同心圆+扇形辐射”**结构，越远离中心/出生点，资源越高级，灵兽越强。此设计强制玩家建立**分基地**。

### Zone 1: 起源平原 (Origin Plains) [Lv1-10]

- **资源:** 木头、石头、浆果、羊毛。
- **核心目标:** 建立第一座木头基地，抓捕`皮皮鸡`和`捣蛋猫`解决温饱。
- **解锁:** 基础工作台、木棒、帕鲁球。

### Zone 2: 迷雾湿地 (Misty Swamp) [Lv10-25]

- **环境:** **毒气**（需防毒面具）。
- **资源:** **原油** (塑料原料)、药草、毒腺、橡胶。
- **核心目标:** 捕捉`森之猩`进行伐木自动化，捕捉`幻影蛾`制作药品。
- **解锁:** 枪械制造台、初级枪械、制药台。

### Zone 3: 锈蚀峡谷 (Rusted Canyon) [Lv25-40]

- **环境:** **干燥/高低落差**。
- **资源:** **金属矿** (核心工业原料)、硫磺、煤炭。
- **核心目标:** 建立**分基地**（矿场）。必须捕捉`碎岩龟`和`熔炉兽`，实现“自动采矿->自动冶炼”流水线。
- **解锁:** 改善型熔炉、金属流水线、手雷。

### Zone 4: 永冻雪峰 (Permafrost Peaks) [Lv35-45]

- **环境:** **极寒**（需防寒服/火系宠）。
- **资源:** **纯水晶** (电路板原料)、蓝宝石。
- **核心目标:** 捕捉`雪怪`和`吹雪狐`，制造冰箱和空调，稳定高级流水线。
- **解锁:** 冰箱、空调、突击步枪、电路板。

### Zone 5: 炼狱火山口 (Inferno Crater) [Lv45+]

- **环境:** **岩浆**（即死/极热，需耐热服）。
- **资源:** **黑曜石**、高能硫磺。
- **核心目标:** 挑战火系神兽，获取顶级装备图纸。只有飞行坐骑能到达中心。
- **解锁:** 传说级装备、火箭筒。

## 3.4 家园守护塔防系统 (Base Defense: The Purge)

区别于Clash of Clans的异步被打，这是**实时PVE**玩法，检验玩家的基地布局和灵兽培养。

### 3.4.1 触发机制

- **被动袭击:** 随着基地物资囤积（金币/稀有蛋），右上角“威胁值”增加。满条后，触发野生灵兽群攻城。
- **主动挑战 (图腾战):** 玩家建造“守护图腾”，献祭特定物品（如Boss掉落物），主动召唤强力怪物波次。胜利可得传说图纸。

### 3.4.2 核心玩法：属性防御塔 (Elemental Turrets)

玩家建造防御塔后，必须**将灵兽“塞进”塔里**才能激活塔的功能（这是将宠物玩法与塔防结合的关键点）：

- **火焰喷射塔:** 放入`[火系]`灵兽 -> 塔喷射扇形火焰（AOE）。
- **雷击塔:** 放入`[雷系]`灵兽 -> 塔发射连锁闪电（麻痹/减速）。
- **加特林机枪:** 放入`[手工作业Lv2+]`灵兽 -> 物理射速极快。
- **投石机:** 放入`[搬运Lv3]`灵兽 -> 投掷巨石，破甲。

### 3.4.3 建筑策略 (Maze Building)

- 怪物AI会寻找通往“帕鲁终端”的最短路径。
- 玩家需利用墙壁构建**“杀戮迷宫 (Kill Box)”**，迫使怪物走S型路线，并经过所有陷阱（地刺、弹射板、毒气喷口）。
- **防空需求:** 后期会有飞行怪直接空降偷家，迫使玩家在屋顶布置防空炮。

### 3.4.4 失败与损失

- **失败:** 终端被毁，基地内部分资源被掠夺，所有灵兽进入“晕厥”状态（需10分钟恢复）。
- **保护:** 基地建筑不会被彻底删除，只会变成“破损状态”（一键消耗材料修复），避免过强的挫败感。

### 3.5 “物质枪”与交互系统 (Multi-Tool Interaction)

为了简化Roblox/手机用户的操作，整合所有工具为一个“物质枪/手套”：

- **采集模式：** 射击方块进行破坏/采集。
- **捕捉模式：** 射出“数据网”捕捉虚弱的灵兽。
- **建造模式：** 类似Minecraft创造模式，射线指哪造哪，支持**网格吸附 (Grid Snapping)**。
- **搬运模式：** 物理吸起一只灵兽或箱子，然后在空中甩出去（基于物理引擎）。

### 3.6 自动化建造系统 (Automation Building)

区别于传统的建造，强调**“逻辑连接”**而非复杂的电线铺设。

- **范围供电：** 发电机产生一个球形“能量场”，范围内的机器自动通电。
- **实体物流：** 物品在传送带上是**真实的物理模型**。这不仅看着爽，而且当传送带堵塞、物品散落一地时，也是一种有趣的物理交互体验。

### 3.7 UGC道馆编辑器 (Dungeon Maker) — 可选

这是后期的核心。提供简单的**逻辑门 (Logic Gates)**：

- **触发器：** 压力板、红外线传感器、怪物死亡触发。
- **执行器：** 门开启/关闭、发射陷阱、生成怪物波次、播放音效。
- **示例：** 玩家可以设计一个“只有在10秒内击败3只火狐狸，大门才会打开”的关卡。

## 第四章：技术架构与开发建议 (Technical & Development)

### 4.1 引擎选型

- **推荐：Unity 2022+ LTS**
    - **理由：** C#开发效率高，Asset Store有大量成熟的体素(Voxel)插件和库存系统，且在移动端性能优化方面比UE5更成熟（考虑到Roblox受众大量使用手机）。

### 4.2 关键技术难点与解决方案

**1. 动态地形下的AI寻路 (NavMesh in Dynamic World)**

- **难点：** 玩家可以随时造墙、挖坑，传统的预烘焙NavMesh无法使用。
- **方案：**
    - 使用 **NavMeshSurface** 进行局部动态烘焙（Runtime Baking）。
    - **防卡死机制：** 必须写一套强力的“脱困逻辑”。如果AI试图移动但坐标未变超过5秒，直接瞬移（传送特效）到目标工作点或回收到终端。**不要试图解决所有寻路bug，要学会规避。**

**2. 海量实体性能优化 (Performance)**

- **难点：** 后期基地可能有100只怪+500个传送带上的物品。
- **方案：**
    - **GPU Instancing：** 对同类物品（如矿石、墙壁）使用GPU实例化渲染。
    - **LOD for Logic (逻辑层LOD)：** 玩家视野外的灵兽，**停止渲染，停止物理运算，只进行纯数值模拟**（产出 = 效率 * 时间）。

### 4.3 开发阶段规划 (Roadmap)

**阶段一：原型验证 (MVP) - 2个月**

- **目标：** 验证核心循环“抓捕->自动化”。
- **产出：**
    - 灰盒地图。
    - 角色3C（移动跳跃）。
    - 1种灵兽（可战斗、可砍树）。
    - 基础建造（放箱子，灵兽自动把木头搬进箱子）。

**阶段二：垂直切片 (Vertical Slice) - 4个月**

- **目标：** 确定美术风格，单机闭环跑通。
- **产出：**
    - 完整的新手村岛屿。
    - 10种灵兽。
    - 完整的UI交互。
    - 手机端操作适配。

**阶段三：量产开发 (Production) - 8个月**

- **目标：** 填充内容。
- **产出：**
    - 4大生态群系（平原、火山、雪地、沙漠）。
    - 60+种灵兽。
    - 联机功能（P2P或专用服务器）。

## 7.2 用户体验剧本 (User Journey Script)

开发团队需确保以下 **15 分钟** 的流程绝对顺畅且有趣：

1. **苏醒 (0-3min)：** 玩家出生，用物质枪手撸大树，获得木材，制造 [工作台] 和 5个 [帕鲁球]。
2. **第一只宠 (3-6min)：** 发现野生 `矿工鼠`。攻击至残血，投掷帕鲁球抓捕。
3. **物理自动化 (6-10min)：**
    - 玩家建造 [采石场]。
    - **关键交互：** 玩家按 F 键将 `矿工鼠` 举过头顶，走到采石场前，瞄准，**用力扔出去**！
    - **反馈：** 矿工鼠落地翻滚，头顶冒出“💡”气泡，开始旋转挖掘。
    - **搬运验证：** 玩家再抓一只 `绵绵球` 扔在矿场边，它自动把地上的石头搬进 [箱子]。
4. **塔防高潮 (10-15min)：**
    - 系统提示“怪物即将以此地为巢穴”。
    - 玩家快速抓捕一只 `火绒狐`。
    - 建造一座 [火焰喷射塔] (此时是熄灭状态)。
    - **核心High点：** 玩家举起 `火绒狐`，按 E 键把它**塞进防御塔的笼子里**。防御塔瞬间喷出火苗，狐狸在里面兴奋地叫。
    - 按下 [召唤] 按钮。一波僵尸冲过来，被防御塔自动识别并喷火烧成灰烬。

## 7.3 系统开发指南 (Technical Guidelines)

### 7.3.1 核心架构：Data-Driven & FSM

- **数据驱动 (ScriptableObject):**
    - 不要把灵兽属性写死。建立 `PetData` 配置文件（包含 ID、模型预制体、工作技能等级、元素类型）。
- **AI 状态机 (Simplified FSM):**
    - 使用简单的 `Enum` 状态切换，无需复杂的行为树插件。
    - `State_Idle`: 待机/游荡。
    - `State_Work`: 播放工作动画 -> 产出掉落物。
    - `State_Hungry`: 寻找 `FeedBox`。
    - **`State_TurretMode` (关键):** 物理运动学关闭 (IsKinematic=true)，模型吸附在塔座上，只旋转朝向敌人。

### 7.3.2 物理交互系统 (Physics Interaction)

**这是区别于其他卡牌类手游的关键。**

- **投掷：** 使用 `Rigidbody.AddForce` 抛出灵兽。
- **碰撞检测：**
    - 灵兽是个刚体。当它碰撞到 `Tag: WorkStation` (如采石场) 时，触发 `AssignWork()`。
    - 当它碰撞到 `Tag: Ground` 时，播放眩晕动画 2 秒，然后恢复 Idle。

### A. 防御塔挂载逻辑 (TurretStation.cs)

`using UnityEngine;

public class TurretStation : MonoBehaviour, IInteractable
{
    [Header("Settings")]
    public Transform seatPoint; // 灵兽坐的位置
    public Transform muzzlePoint; // 发射火球的位置
    public GameObject projectilePrefab;
    public float fireRate = 1.0f;
    
    private PetController assignedPet; // 当前塔里的灵兽
    private float nextFireTime;

    // 1. 交互逻辑：玩家按E把手里的怪塞进去
    public void OnInteract(PlayerController player)
    {
        if (player.heldPet != null)
        {
            // 验证核心逻辑：只有火系才能进火焰塔
            if (player.heldPet.data.elementType == ElementType.Fire)
            {
                MountPet(player.heldPet);
                player.ClearHand(); // 清空玩家手持
            }
            else
            {
                UI.ShowTip("这只灵兽不会喷火！需要火系灵兽！");
            }
        }
    }

    void MountPet(PetController pet)
    {
        assignedPet = pet;
        
        // 物理处理：禁用寻路，禁用重力，吸附到座位
        pet.agent.enabled = false; 
        pet.rb.isKinematic = true; 
        pet.transform.SetParent(seatPoint);
        pet.transform.localPosition = Vector3.zero;
        pet.transform.localRotation = Quaternion.identity;
        
        // 切换AI状态
        pet.SetState(PetState.TurretMode);
        Debug.Log("防御塔激活！");
    }

    // 2. 自动射击逻辑
    void Update()
    {
        if (assignedPet == null) return;

        GameObject target = FindClosestEnemy(); // 简单的球形射线检测
        if (target != null)
        {
            // 塔身旋转 (带动灵兽一起转)
            Vector3 dir = target.transform.position - transform.position;
            Quaternion lookRot = Quaternion.LookRotation(dir);
            transform.rotation = Quaternion.Lerp(transform.rotation, lookRot, Time.deltaTime * 5f);

            if (Time.time > nextFireTime)
            {
                Fire(target);
                nextFireTime = Time.time + fireRate;
            }
        }
    }

    void Fire(GameObject target)
    {
        // 调用灵兽的攻击动画
        assignedPet.animator.SetTrigger("Attack");
        // 生成子弹
        Instantiate(projectilePrefab, muzzlePoint.position, muzzlePoint.rotation);
    }
}`

{
    if (!isThrown) return; // 只在被扔出去时检测

    // 1. 砸到了工作台
    if (collision.gameObject.TryGetComponent(out IWorkStation station))
    {
        // 检查技能匹配度 (比如：拿火狐狸砸矿场是无效的)

## 7.6 验收标准 (Success Criteria)

在 MVP 评审会上，演示必须通过以下测试：

1. **物理测试：** 举起一只猫扔向墙壁，它应该反弹并晕倒，而不是穿墙或卡住。
2. **自动化测试：** 放置一只 [矿工鼠] 和一只 [绵绵球]，不进行任何操作，5分钟后箱子里应该自动出现 50+ 个石头。
3. **塔防测试：** **(关键)** 必须清晰展示“狐狸进塔 -> 塔喷火”的过程。如果这个过程看起来很滑稽、很有趣（例如狐狸在塔里拼命喷火的样子），则 MVP 成功。
